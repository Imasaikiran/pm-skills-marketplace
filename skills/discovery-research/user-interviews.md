# User Interviews Skill

## Description
Conduct effective user interviews to gather qualitative insights and validate product assumptions.

## When to Use
- Discovering user needs and pain points
- Validating product ideas before development
- Understanding user workflows and behaviors
- Gathering feedback on existing features

## Interview Preparation

### Define Objectives
1. What do you want to learn?
2. What assumptions are you testing?
3. What decisions will this inform?

### Create Discussion Guide
- Opening questions (warm-up)
- Core questions (main insights)
- Follow-up probes
- Closing questions

### Recruit Participants
- Define target user segments
- Aim for 5-8 interviews per segment
- Screen for relevant characteristics
- Offer incentives if appropriate

## Interview Best Practices

### Before the Interview
- [ ] Research participant background
- [ ] Test recording equipment
- [ ] Prepare discussion guide
- [ ] Set up note-taking system

### During the Interview
- [ ] Build rapport and set expectations
- [ ] Ask open-ended questions
- [ ] Listen more than you talk (80/20 rule)
- [ ] Probe deeper with "why" questions
- [ ] Avoid leading questions
- [ ] Observe non-verbal cues
- [ ] Take detailed notes

### After the Interview
- [ ] Review and organize notes
- [ ] Identify key themes
- [ ] Share insights with team
- [ ] Update product assumptions

## Effective Questions

### Good Opening Questions
- "Tell me about the last time you [used X product/did Y task]"
- "Walk me through your typical [workflow/day]"
- "What challenges do you face when [doing X]?"

### Good Follow-Up Questions
- "Can you tell me more about that?"
- "Why is that important to you?"
- "How did that make you feel?"
- "What would you do differently?"

### Questions to Avoid
- ❌ "Would you use this feature?" (hypothetical)
- ❌ "Don't you think this is a good idea?" (leading)
- ❌ "Would you pay for this?" (unreliable)

## Analysis Framework

### Identify Patterns
- Common pain points
- Recurring behaviors
- Shared goals
- Workflow similarities

### Extract Insights
- User needs vs. wants
- Workarounds and hacks
- Emotional responses
- Unexpected findings

### Document Findings
- Key quotes
- Behavior patterns
- Opportunity areas
- Product implications

## Common Pitfalls

1. **Asking leading questions** - Let users tell their story
2. **Talking too much** - Listen actively
3. **Taking answers at face value** - Probe deeper
4. **Interviewing the wrong people** - Screen carefully
5. **Not recording** - Don't rely on memory alone
6. **Ignoring contradictions** - These reveal important insights

## Tips for Success

- **Be curious** - Genuine interest shows
- **Stay neutral** - Don't show bias
- **Embrace silence** - Give users time to think
- **Follow the energy** - Explore topics that excite users
- **Look for stories** - Past behavior predicts future behavior
- **Build trust** - Start with easy questions
