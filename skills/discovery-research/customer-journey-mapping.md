# Customer Journey Mapping

## Description
Customer Journey Mapping is a visualization technique that illustrates the complete experience a customer has with your product or service across all touchpoints, channels, and time periods, revealing pain points, opportunities, and moments that matter.

## When to Use
- Understanding end-to-end customer experience
- Identifying pain points and friction
- Finding opportunities for improvement
- Aligning teams on customer perspective
- Designing new features or products
- Improving specific stages (onboarding, conversion, etc.)
- Building empathy across organization
- Prioritizing UX improvements

## Journey Map Components

### Core Elements

#### 1. Persona/Actor
Who is experiencing this journey

#### 2. Scenario
What goal or job they're trying to accomplish

#### 3. Phases/Stages
Major steps in the journey

#### 4. Actions
What customer does at each stage

#### 5. Touchpoints
Where interactions occur

#### 6. Emotions
How customer feels throughout

#### 7. Pain Points
Frustrations and obstacles

#### 8. Opportunities
Areas for improvement

---

## Journey Map Types

### 1. Current State Journey
**Purpose:** Understand existing experience

**Use When:**
- Auditing current experience
- Finding pain points
- Identifying quick wins
- Building empathy

**Example Scenario:** "New user signs up and completes first task"

---

### 2. Future State Journey
**Purpose:** Envision improved experience

**Use When:**
- Designing new features
- Planning improvements
- Setting vision
- Communicating strategy

**Example Scenario:** "New user experiences frictionless onboarding in redesigned flow"

---

### 3. Day-in-the-Life Journey
**Purpose:** Understand broader context

**Use When:**
- Entering new market
- Understanding user context
- Finding integration opportunities
- Discovering jobs to be done

**Example Scenario:** "A day in the life of a marketing manager"

---

### 4. Service Blueprint
**Purpose:** Include behind-the-scenes processes

**Use When:**
- Involving operations teams
- Planning implementation
- Understanding dependencies
- Improving service delivery

**Includes:**
- Front-stage actions (visible to customer)
- Back-stage actions (not visible)
- Support processes
- Systems and tools

---

## Step-by-Step Process

### Phase 1: Prepare (Before Workshop)

**1. Define Scope**
- Which persona?
- Which scenario/goal?
- Time frame to cover?
- Phases to include?

**2. Gather Research**
- User interviews
- Support tickets
- Analytics data
- Session recordings
- Surveys
- Sales conversations

**3. Assemble Team**
- Product managers
- Designers
- Engineers
- Customer success
- Sales
- Marketing

**4. Choose Format**
- Physical board
- Digital tool (Miro, Mural, FigJam)
- Template to use

---

### Phase 2: Map the Journey (Workshop)

**Duration:** 2-3 hours

**Step 1: Set Context (15 min)**
- Introduce persona
- Share research insights
- Explain the scenario
- Review objectives

**Step 2: Define Phases (15 min)**
- Identify major stages
- Agree on phase names
- Determine phase sequence
- Set phase boundaries

**Step 3: Map Actions (30 min)**
- What does customer do at each phase?
- List all actions
- Use sticky notes
- One action per note

**Step 4: Identify Touchpoints (20 min)**
- Where do actions happen?
- What channels are used?
- Which teams are involved?
- Map to each action

**Step 5: Plot Emotions (20 min)**
- How does customer feel?
- Create emotional arc
- Use emotion scale (-5 to +5)
- Draw line graph

**Step 6: Highlight Pain Points (20 min)**
- Where is friction?
- What frustrates users?
- What causes abandonment?
- Mark with red dots

**Step 7: Add Opportunities (20 min)**
- What could be improved?
- Where can we add value?
- Quick wins vs long-term
- Mark with green dots

**Step 8: Identify Metrics (15 min)**
- How to measure each phase?
- What indicates success?
- Where are drop-offs?

---

### Phase 3: Analyze & Prioritize

**1. Identify Themes**
- Common pain points
- Repeated issues
- Critical moments
- Emotional peaks/valleys

**2. Assess Impact**
- How many users affected?
- How severe is pain?
- Impact on key metrics?
- Strategic importance?

**3. Estimate Effort**
- Design complexity
- Engineering work
- Dependencies
- Time to implement

**4. Prioritize Opportunities**
- Use RICE or other framework
- Consider quick wins
- Focus on critical paths
- Balance short and long-term

## Emotion Mapping Techniques

### Emotion Scale

**+5:** Delighted, thrilled, amazed
**+4:** Very happy, satisfied
**+3:** Happy, pleased
**+2:** Somewhat pleased
**+1:** Slightly positive
**0:** Neutral
**-1:** Slightly negative
**-2:** Somewhat frustrated
**-3:** Frustrated, disappointed
**-4:** Very frustrated, angry
**-5:** Extremely frustrated, rage

---

### Emotional Arc Patterns

**1. The Roller Coaster**
- High excitement at start
- Deep frustration in middle
- Recovery at end
- **Action:** Reduce middle friction

**2. The Gradual Decline**
- Starts positive
- Slowly degrades
- Never recovers
- **Action:** Identify wearing factors

**3. The Steady Climb**
- Starts uncertain
- Builds confidence
- Ends positive
- **Action:** Maintain momentum

**4. The Plateau**
- Quick positive spike
- Long neutral period
- Risk of churn
- **Action:** Add engagement moments

## Workshop Facilitation Tips

### Preparation

**1. Set Clear Objectives**
- What will we do with the map?
- What decisions will it inform?
- What's in/out of scope?

**2. Share Research in Advance**
- User quotes
- Data highlights
- Key findings
- Pain point summaries

**3. Create Safe Environment**
- No bad ideas
- All perspectives welcome
- Focus on customer, not blame
- Time-box discussions

---

### During Workshop

**1. Start with Stories**
- Share real customer quotes
- Watch user videos
- Read support tickets
- Build empathy first

**2. Use Facilitator Techniques**
- Park off-topic ideas
- Keep moving forward
- Encourage quiet voices
- Synthesize as you go

**3. Make it Visual**
- Use colors
- Draw emotions
- Add icons
- Include photos

**4. Stay Customer-Focused**
- Use "I" statements (as customer)
- Reference real examples
- Challenge assumptions
- Verify with data

---

### After Workshop

**1. Digitize and Polish**
- Clean up the map
- Add visual design
- Include photos/screenshots
- Make it presentable

**2. Validate Findings**
- Review with customers
- Check against data
- Get stakeholder input
- Test assumptions

**3. Share Widely**
- Present to leadership
- Post in shared space
- Create summary deck
- Tell the story

**4. Create Action Plan**
- Prioritized opportunities
- Owners assigned
- Timeline set
- Success metrics defined

## Common Mistakes to Avoid

### 1. Creating Journey in Isolation
**Problem:** Map doesn't reflect reality
**Solution:** Include cross-functional team and base on research

### 2. Too Much Detail
**Problem:** Map becomes overwhelming
**Solution:** Focus on key moments, create separate detailed views

### 3. Making It Pretty Instead of Useful
**Problem:** Spending time on design, not insights
**Solution:** Start rough, polish only for final sharing

### 4. Only Mapping Happy Path
**Problem:** Missing edge cases and failures
**Solution:** Include error states, edge cases, failures

### 5. Creating Once and Forgetting
**Problem:** Map becomes outdated
**Solution:** Review and update quarterly

### 6. Not Connecting to Action
**Problem:** Pretty map, no changes
**Solution:** Define clear next steps and owners

### 7. Internal Perspective
**Problem:** Mapping what we think, not what customers experience
**Solution:** Ground in real research and customer voice
